# OpenBiliClaw v0.1 MVP — 开发任务清单

> **目标**：证明"深度理解用户 → 主动发现内容 → 有温度地推荐"这条链路是通的。
>
> **交付物**：一个可在本地运行的 CLI 程序 + Chrome 插件，能从 B 站获取用户历史，构建初步用户画像，搜索并推荐内容，并收集反馈。

---

## 开发优先级说明

- **P0** — 阻塞核心流程，必须完成
- **P1** — 重要功能，v0.1 应包含
- **P2** — 可选增强，时间允许则加入

> **验收**：SocraticDialogue 通过 LLMService 调用 LLM，prompt 中自动包含用户画像；provider 错误和空响应均有优雅处理；所有 prompt 组装逻辑集中在 `prompts.py` 而非散落各处。

## 3. Bilibili 接入层 (P0)

### 3.1 Cookie 认证

- [ ] 实现 `AuthManager.set_cookie()`：接收用户粘贴的 cookie 字符串
- [ ] 实现 cookie 持久化到 `data/bilibili_cookie.json`（加密存储，P2）
- [ ] 实现 cookie 有效性验证：调用 B 站导航 API 检查登录态
- [ ] CLI 命令 `openbiliclaw auth login`：引导用户设置 cookie
- [ ] CLI 命令 `openbiliclaw auth status`：显示当前认证状态

> **验收**：用户执行 `openbiliclaw auth login`，粘贴 cookie，程序确认登录成功并显示用户昵称；重启后自动加载已保存的 cookie。

### 3.2 核心 API 实现

- [ ] `get_user_history()`：获取用户观看历史（分页，至少拉取 200 条）
- [ ] `get_favorites()`：获取用户收藏夹列表 + 收藏内容
- [ ] `get_following()`：获取关注的 UP 主列表
- [ ] `search()`：关键词搜索，支持分页和排序参数
- [ ] `get_video_info()`：获取单个视频详情
- [ ] `get_related_videos()`：获取相关推荐视频
- [ ] `get_ranking()`：获取分区排行榜
- [ ] `get_video_comments()`：获取视频评论（前 N 条热评）
- [ ] 统一的错误处理：网络错误、风控限制、cookie 过期等
- [ ] 请求限流：添加请求间隔控制，避免触发 B 站反爬

> **验收**：每个 API 方法有对应的单元测试（mock HTTP）和一个可选的集成测试（真实请求，需 cookie，标记为 `@pytest.mark.integration`）。

### 3.3 agent-browser 集成 (P1)

- [ ] 检测 agent-browser 是否安装，未安装时给出安装提示
- [ ] 实现 `navigate()` + `get_page_content()` 基础操作
- [ ] *(P2)* 实现评论区抓取等 API 无法完成的操作

> **验收**：在 agent-browser 已安装的环境下，能通过浏览器打开 B 站页面并获取内容。

## 4. 记忆系统 (P0)

### 4.1 事件层 (Event Layer)

- [ ] 实现事件写入 SQLite（`events` 表）
- [ ] 实现事件查询：按时间范围、类型、关键词筛选
- [ ] 定义事件类型枚举：`view`, `search`, `favorite`, `like`, `comment`, `click`, `feedback`
- [ ] 实现事件统计：某时间段内各类型事件数量

> **验收**：调用 `memory_manager.propagate_event(event)` 后，事件写入数据库；能按时间和类型查询。

### 4.2 偏好层 (Preference Layer)

- [ ] 定义偏好数据结构（对应 `profile.py` 中的 `PreferenceLayer`）
- [ ] 实现从事件层**批量分析**生成偏好：调用 LLM 分析最近 N 条事件，提取兴趣标签
- [ ] 实现标签权重衰减：随时间降低旧标签权重
- [ ] 偏好层数据持久化为 JSON 文件 (`data/memory/preference.json`)

> **验收**：给定一批用户事件数据，LLM 能提取出 5~15 个兴趣标签（带权重），存储到 JSON 文件；第二次运行时能加载并增量更新。

### 4.3 灵魂层 (Soul Layer)

- [x] 实现初始画像生成：从用户历史 + 偏好层生成 `SoulProfile`
- [x] 画像包含：人格描述（自然语言）、核心特质、深层需求推断
- [x] 画像持久化为 JSON (`data/memory/soul.json`)
- [x] 实现 `to_llm_context()`：生成可注入 LLM prompt 的画像摘要
- [x] CLI 命令 `openbiliclaw profile`：展示当前画像（Rich 格式化输出）

> **验收**：首次运行时从 B 站历史生成一段 200+ 字的人格描述和 3~5 个核心特质；CLI 展示画像时排版美观可读。

### 4.4 觉察层 + 洞察层 (P2，v0.1 可简化)

- [x] *(P2)* 实现每日觉察笔记生成
- [x] *(P2)* 实现洞察假设管理（创建/验证/调整）

> v0.1 中觉察层和洞察层可以简化为空，核心先跑通 事件→偏好→灵魂 三层。

### 4.5 核心记忆加载

- [x] 实现 `MemoryManager.get_core_memory()`：返回 Soul + Preference 摘要
- [x] 所有需要 LLM 的模块，调用 LLM 前自动注入 core memory 作为 system prompt 的一部分
- [x] 实现 `MemoryManager.initialize()` 和 `save_all()`

> **验收**：在 Agent 的任何 LLM 调用中，能在 prompt 中看到用户画像信息。

## 5. 内容发现引擎 (P0)

### 5.1 搜索策略 (SearchStrategy)

- [ ] 实现 LLM 关键词生成：给定用户画像，生成 5~10 个搜索关键词组合
- [ ] 对每个关键词调用 `BilibiliAPIClient.search()`
- [ ] 对结果进行去重（bvid 去重）
- [ ] 对每个结果生成 `DiscoveredContent` 对象

> **验收**：给定一个用户画像，能自动生成搜索词并返回 20+ 条去重后的内容。

### 5.2 排行榜策略 (TrendingStrategy) (P1)

- [ ] 获取各相关分区的排行榜
- [ ] 用 LLM 评估每条内容与用户画像的匹配度（打分 0~1）
- [ ] 只保留得分 > 阈值的内容

> **验收**：能自动拉取排行榜数据并筛选出与用户匹配的内容。

### 5.3 相关推荐链策略 (RelatedChainStrategy) (P1)

- [ ] 从用户最近高评价/收藏的视频出发
- [ ] 调用 `get_related_videos()` 获取相关推荐
- [ ] 递归 2~3 层（控制深度，避免发散太远）
- [ ] 对每条评估匹配度

> **验收**：从一个种子视频出发，能沿推荐链发现 10+ 条新内容。

### 5.4 跨领域探索策略 (ExploreStrategy) (P2)

- [ ] *(P2)* LLM 基于画像推断潜在新兴趣领域
- [ ] *(P2)* 在新领域搜索并推荐

### 5.5 内容评估

- [ ] 实现 `evaluate_content()`：用 LLM 评估内容与用户 Soul 的匹配度
- [ ] Prompt 设计：提供用户画像 + 视频信息（标题/标签/描述/UP主），要求打分 + 给出理由
- [ ] 评估结果写入 `DiscoveredContent.relevance_score` 和 `relevance_reason`

> **验收**：评估 10 条内容，每条有 0~1 的分数和一句理由。

### 5.6 发现引擎编排

- [ ] 实现 `ContentDiscoveryEngine.discover()`：并行执行多个策略
- [ ] 合并、去重、排序结果
- [ ] 将发现的内容缓存到 SQLite (`content_cache` 表)

> **验收**：一次发现周期能从多个策略收集 30+ 条去重内容，按匹配度排序。

## 6. 推荐引擎 (P0)

### 6.1 推荐排序

- [ ] 从 content_cache 中选取 Top-N 未推荐过的内容
- [ ] 去除已推荐过的内容（查询 `recommendations` 表）
- [ ] 按 relevance_score 排序

### 6.2 朋友式推荐表达

- [ ] 实现 `generate_expression()`：为每条推荐生成朋友式推荐理由
- [ ] Prompt 设计要点：
  - 注入完整用户画像作为上下文
  - 要求输出像朋友推荐一样的自然语言（带"我觉得"、"我理解你"）
  - 解释为什么这个**人**会喜欢这个**内容**（不是"因为你看过类似的"）
  - 中文输出，口语化，有温度
- [ ] 每条推荐表达 50~150 字

> **验收**：生成的推荐文案读起来像朋友在微信给你发消息推荐视频，不像算法推荐。示例：*"我觉得你会喜欢这个——这个 UP 主讲历史的方式特别对你胃口，就是那种深入原理的风格。"*

### 6.3 推荐持久化

- [ ] 推荐记录写入 `recommendations` 表
- [ ] 记录推荐时间、是否展示、用户反馈

### 7.1 核心命令

|------|------|--------|
| `openbiliclaw auth login` | 设置 B 站 cookie | P0 |

- [ ] 用 Rich 库实现美观的终端输出

- [ ] 用户画像：分区块展示（人格描述、兴趣标签、核心特质）
- [ ] 操作反馈有颜色和 emoji

> **验收**：终端输出排版美观，信息层次清晰，非技术用户也能看懂。

## 9. 反馈循环 (P1)

### 9.1 反馈处理

- [ ] CLI 反馈命令：`openbiliclaw feedback <推荐ID> <like|dislike|comment>`
- [ ] 反馈写入 `recommendations` 表
- [ ] 反馈同时写入事件层（作为 `feedback` 类型事件）

### 9.2 画像更新

- [ ] 收集到 N 条新反馈后，触发偏好层重新分析
- [ ] 偏好层变化超过阈值时，触发灵魂层更新
- [ ] 更新后的画像持久化

> **验收**：用户连续对几条推荐给出"不喜欢"反馈后，下次推荐的内容方向有明显变化。
